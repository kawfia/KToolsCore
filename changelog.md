## [2.0.8] — beta

Чекбокс «Закрывать лут при снятии шкур». При начале каста (spell ID 8613) и открытом окне лута — вызывает `CloseLoot()`. После завершения каста открывается окно со шкурой.

## [2.0.7] — beta

### [bug-027] Пустые слоты в окне лута после подбора
Откат с LOOT_READY на LOOT_OPENED: при LOOT_READY UI строился после LootSlot() и показывал уже пустые слоты. LOOT_OPENED гарантирует правильное состояние фрейма.

### [bug-028] Серые предметы не собирались при фильтре «Плохое» качество
Убрано ограничение `IsEquipment()` из фильтра качества — теперь фильтр применяется ко всем предметам, а не только к оружию/броне.

### [bug-029] Очередь вскрытия токенов запускается после закрытия окна лута
`ProcessBagTokens()` вызывается из `OnLootClosed` (с задержкой 0.2s), открывает по одному токену, пересобирает себя через 0.5s пока есть ещё. Убран `OnBagUpdate` и `BAG_UPDATE` регистрация. Исправлена константа `NUM_BAG_SLOTS` → `NUM_BAG_FRAMES`.

### [bug-030] Дропдаун удаления сохранял выбор после отказа
`refreshDeleteDropdown()` вызывается в `OnCancel` — сбрасывает выбор и пересобирает список.

## [2.0.6] — beta

### [ref-035] Пустое окно добычи закрывается мгновенно
Если `GetNumLootItems()` возвращает 0 при включённом аддоне — вызывается `CloseLoot()` немедленно.

### [ref-036] Переход с LOOT_OPENED на LOOT_READY
`LOOT_READY` срабатывает до отрисовки UI окна добычи. Лут подбирается до появления окна у игрока. Аналог механики Leatrix Plus.

### [ref-032] Повторная попытка подбора предметов с незакэшированными данными
Слоты где `GetItemInfo` вернул `classID=nil` помещаются в очередь. При событии `GET_ITEM_INFO_RECEIVED` — повторная проверка и `LootSlot()`. `LOOT_CLOSED` очищает очередь.

## [2.0.5] — beta

### [new-002] Сила артефакта подбирается по classID без внешней библиотеки
Удалена LibArtifactPower-ItemID-1.0. Определение через `classID=0, subClassID=8` (Consumable/Other). Существующий чекбокс «Сила артефакта» теперь работает.

### [ref-033] Галочка «Контейнеры» переименована в «Токены»
Охватывает classID=1 (сумки), classID=15/sub=0 (Legion лут-мешки) и classID=15/sub=4 (Legion токены вроде «Добычи кровавого охотника»). Профильный ключ: `containers` → `tokens`.

### [ref-034] Вскрытие токенов/контейнеров из сумки переработано
Убран нерабочий механизм очереди (`EnqueueContainer`). Прямое сканирование сумок при `BAG_UPDATE` — использует `UseContainerItem` по одному за раз, следующий `BAG_UPDATE` подбирает следующий. Охватывает те же classID что и галочка «Токены».

## [2.0.3] — beta

### [bug-024] Оба дропдауна профилей перестраивают список при каждом открытии
Активный профиль и удаление — список строится динамически через HookScript("OnClick"). После удаления выбор в дропдауне удаления сбрасывается.

### [bug-025] IO диалог закрывается вместе с главным окном; импорт делает корректный cleanup
Главное окно при закрытии вызывает `ioDialog:Hide()`. Кнопка «Импортировать» использует `dlg:Hide()` вместо `AceGUI:Release()` для правильного срабатывания OnClose.

### [bug-026] Legion лут-мешки (classID=15, subclass=0) теперь подбираются
Добавлен `IsContainer()` — проверяет classID=1 (стандартные сумки) и classID=15/sub=0 (Emissary Cache и другие Legion контейнеры).
