# Архитектура — Правила

## Ядро и модули
Ядро **не знает** о модулях. Модули сами регистрируются в ядре при загрузке.
Ядро предоставляет: функцию регистрации, безопасный вызов, единый UI-контейнер.

```lua
KTools:RegisterModule(name, module)   -- вызывается из модуля при загрузке
KTools:SafeCall(module, method, ...)  -- ядро вызывает методы модулей
```

Критическая ошибка в одном модуле не роняет остальные.

---

## Хардкод запрещён
Никаких жёстко зашитых значений в коде кроме .toc файла: имя разработчика, версия аддона, название аддона.
Всё берётся из `.toc` файла или SavedVariables.

---

## Локализация
В каждом аддоне своя локаль. Структура:

```
locale/
  load.xml    — список файлов локали
  ruRU.lua
  enUS.lua
```

`load.xml` содержит ссылки на `.lua` файлы (локали, core-файлы).
`load.xml` находится только в подпапках и описывает нижележащие файлы.
На `load.xml` ссылается `.toc` файл аддона.

---

## Структура файлов
`.toc` → `load.xml` → `.lua` файлы.

---

## Комментарии в проекте
В каждом файле (кроме .toc) первой строкой обязан быть комментарий — полный путь к файлу начиная с папки аддона:

```lua
-- KTools/core/ui.lua
```

---

## .toc файлы
`## Notes:` — только английский текст.
`## Notes-ruRU:` — русский перевод отдельной строкой.

---

## Файловая структура проекта

```
KTools/                          — ядро
  KTools.toc                     — точка входа аддона
  init.lua                       — инициализация, регистрация Ace
  core/
    load.xml                     — список core-файлов
    minimap.lua                  — кнопка миникарты (LibDBIcon)
    settings.lua                 — SavedVariables ядра
    ui.lua                       — главное окно, навигация, RegisterModule, SafeCall
  lib/
    load.xml                     — подключение библиотек
  locale/
    load.xml                     — список файлов локали
    enUS.lua
    ruRU.lua
  media/
    minimapButton_dx5.blp        — иконка кнопки миникарты

KToolsAutoloot/
  KToolsAutoLoot.toc

KToolsTPH/
  KToolsTPH.toc
```
