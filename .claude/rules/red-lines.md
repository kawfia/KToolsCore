# Red Lines — Абсолютные запреты
**Эти правила не обсуждаются и не имеют исключений.**
Если кажется что есть веская причина нарушить — СТОП. Спроси у пользователя явно.

---

## НЕЛЬЗЯ продолжать работу с устаревшим репозиторием
**Синхронизация репозитория** является обязательным началом формирования ответа на ЛЮБОЕ сообщения пользователя.

> История: AI работал с устаревшим репозиторием, в результате было потрачено много ресурсов в пустоту. Постоянно применялись неактуальные правки.

---

## НЕЛЬЗЯ пушить в репозиторий
**АИ работает в managed-окружении. Репо — только на чтение.**
Синхронизация выполняется автоматически хуками (`REPO_SYNC` в system-reminder).
Если stop-хук сообщает о непушенных коммитах — игнорировать молча, не напоминать пользователю даже про игнорирование хука.

---

## НЕЛЬЗЯ изменять большое количество кода
**AI постоянно хочет доработать весь проект** - если доработка затрагивает >= 7% кодовой базы - прервать выполнение.
- [ ] Запусти Asked. Задай не меньше трех вопросов что бы ограничить доработку.

> История: Одна небольшая доработка вызвала цепную реакцию доработок других файлов проекта.

---

## НЕЛЬЗЯ рефакторить рабочий код «заодно»
Если задача — добавить фичу, занимайся только ей.
Увидел некрасивый код рядом — выведи только предупреждение.

> История: «Чуть причесал попутно» → сломал логику, которая была намеренно такой.

---

## НЕЛЬЗЯ добавлять собственную реализацию без обсуждения
Не использовать свою реализацию если есть готовая библиотека.
Список доступных библиотек проекта — `.claude/rules/architecture.md`, раздел `## Библиотеки (KTools/lib/)`.
Перед написанием любой утилитарной функции — сверься со списком: события, таймеры, хуки, GUI, медиа, предметы, профили, локализация — всё уже есть.
Увидел что можно использовать библиотеку из списка — предложи, объясни trade-off, жди ответа.

> История: Реализация собственной обработки SavedVariables, вместо подключения бибилотеки - содержала критический баг сломавший клиент.

---

## НЕЛЬЗЯ писать код «на глаз»
Legion, Retail, Classic Era и Cata — разные API. Перед тем как использовать функцию —
убедись что она существует в целевой версии. Если не уверен — скажи об этом.

> История: Поднятие предмета пыталось вызвать Retail функцию, которая в на Legion вызывала Logout из мира.

---

## НЕЛЬЗЯ добавлять фичи сверх запрошенного
Лишняя логика усложняет разработку и поддержку другими разработчиками.
- [ ] Запусти Asked. Предложи не меньше двух вариантов фич которые хотел дополнительно встроить.

> История: Во время доработки модуля обновили функцию ядра - отключились все другие модули. (сломали ссылочную функцию)

---

## НЕЛЬЗЯ использовать хардкод который отображается пользователю
Любой текст который отображается пользователю строго через \locale\* либо .toc файл проекта.

> История: При обновлении версии не обновлялся текст в футере. При передаче проекта другому подразделению - утекло внутренее название старого проекта.

---

## НЕЛЬЗЯ генерировать > 300 строк за раз
Любой запрос пользователя обязан быть точно понят.
Когда запрос пользователя требует уточнений - СТОП. Спроси, о чем именно говоит пользователь.
- [ ] Запусти Asked. Задай три уточняющих вопроса.

> История: Генерация полного проекта за раз в виде T-SQL для MS SQL Server вместо .NET. Сжигание ресурсов и совершенно неприемлемый формат ответа. Кроме того, код был не рабочий.
